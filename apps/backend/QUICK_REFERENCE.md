## ⚡ Key Implementation Details

### Size Guide (HTML Table)
```sql
size_guide_html TEXT    -- Stores raw HTML table
-- Example: <table><tr><th>Size</th><th>Bust</th></tr>...
```

### Barcode (Auto-Generated)
```sql
sku VARCHAR(100) UNIQUE  -- Auto-generated by system
-- Pattern: Could be UUID or timestamp-based
-- Indexed for fast lookup
```

### Promotions (Login Required)
```sql
requires_login BOOLEAN DEFAULT true
-- All promotions in DB have requires_login = true
-- Endpoint: GET /promotions/active requires JWT
-- Guard: @UseGuards(JwtAuthGuard)
```

### Promo Codes (Login Required)

## 💡 Tips for Development

### Testing Authentication
```bash
# Register user
POST /api/v1/auth/register
{
  "email": "test@example.com",
  "password": "SecurePass123!",
  "first_name": "John",
  "last_name": "Doe"
}

# Login
POST /api/v1/auth/login
{
  "email": "test@example.com",
  "password": "SecurePass123!"
}
# Returns: { access_token, refresh_token }

# Use JWT
Authorization: Bearer <access_token>
```

### Creating Products
```bash
POST /api/v1/products (admin)
{
  "name": "Product Name",
  "category_id": "uuid",
  "price": 5000,
  "available_sizes": ["S", "M", "L", "XL"],
  "size_guide_html": "<table>...</table>",
  "main_image": "cloudinary_url"
}
# System auto-generates SKU/barcode
```

### Applying Promo Code
```bash
# MUST be logged in (JWT required)
GET /api/v1/promo-codes/PRET30/validate
Authorization: Bearer <token>
# Returns: { valid: true, discount: 15000, ... }
```

### Tracking Product Views
```bash
POST /api/v1/products/:id/track
{
  "event_type": "view",
  "duration_seconds": 30
}
# Increments view_count automatically
```

## 🎓 Key Learnings

### Promotions & Promo Codes
- Both stored in database with `requires_login = true`
- JWT Guard required on validation endpoints
- User cannot apply promotions as guest
- User cannot use promo code as guest
- Per-user usage limits tracked

### Size Management
- HTML table stored in database (not image)
- Flexible format for descriptions
- Admin can customize per product
- Inventory tracked per size

### Barcode
- System auto-generates unique SKU
- Not editable by admin or user
- Can generate barcode image for printing
- Used for POS scanning

### Analytics
- Works for both guest & registered users
- Guest tracked via session_id
- View tracking automatic on endpoint hit
- Extensible for clicks, purchases
